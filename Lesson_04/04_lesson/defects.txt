Название дефекта: Некорректная работа contains() при передаче пустой строки в качестве искомого символа

ID: DEF-001
Приоритет: Средний
Серьёзность: Низкая
Статус: Новый

Описание:
Метод contains(string, symbol) возвращает True, если в качестве symbol передана пустая строка ("").

Шаги воспроизведения:
1. Вызвать contains("SkyPro", "")
2. Получить результат

Фактический результат:
Метод возвращает True.

Ожидаемый результат:
Либо:
- Метод должен возвращать False,
либо:
- Метод должен выбрасывать исключение ValueError,
так как поиск пустого символа логически некорректен.

Комментарий:
Текущее поведение связано с использованием string.index(symbol),
который для пустой строки возвращает 0.
Рекомендуется явно обработать этот кейс в коде.

Название дефекта: trim() удаляет только обычный пробел, но не другие whitespace-символы

ID: DEF-002
Приоритет: Низкий
Серьёзность: Низкая
Статус: Новый

Описание:
Метод trim() удаляет только символ " " (пробел), но не удаляет табуляцию (\t) и перенос строки (\n).

Шаги воспроизведения:
1. Вызвать trim("\t\tSkyPro")
2. Получить результат

Фактический результат:
Строка остаётся без изменений.

Ожидаемый результат:
Метод должен удалять все начальные whitespace-символы,
аналогично стандартному string.lstrip().

Комментарий:
Рекомендуется заменить цикл на использование string.lstrip().

DEF-001

Название: contains() возвращает True при поиске пустой строки

Приоритет: Средний
Серьёзность: Низкая
Статус: Новый

Описание

Метод contains(string, symbol) возвращает True, если symbol == "".

Шаги воспроизведения
utils.contains("SkyPro", "")
Фактический результат

Возвращает True.

Ожидаемый результат

Метод должен:

либо возвращать False

либо выбрасывать ValueError

Причина

string.index("") возвращает 0.

Рекомендация

Добавить явную проверку:

if not symbol:
    raise ValueError("Symbol cannot be empty")
DEF-002

Название: trim() удаляет только обычный пробел

Приоритет: Низкий
Серьёзность: Низкая
Статус: Новый

Описание

Метод удаляет только символ " ", но не обрабатывает \t, \n.

Шаги воспроизведения
utils.trim("\t\tSkyPro")
Фактический результат

Строка не изменяется.

Ожидаемый результат

Все начальные whitespace-символы должны удаляться.

Рекомендация

Использовать:

string.lstrip()
DEF-003

Название: Методы не валидируют входные параметры

Приоритет: Высокий
Серьёзность: Средняя
Статус: Новый

Описание

Методы не проверяют тип входных данных.

Шаги воспроизведения
utils.capitalize(None)
utils.trim([])
utils.contains(123, "1")
Фактический результат

Выбрасываются AttributeError или TypeError.

Ожидаемый результат

Методы должны явно проверять тип аргументов и выбрасывать понятное исключение:

TypeError: string must be str
Бизнес-риск

Неочевидные ошибки усложняют поддержку и отладку.

Рекомендация

Добавить проверку:

if not isinstance(string, str):
    raise TypeError("string must be str")

DEF-005

Название: contains() использует index() вместо оператора in

Приоритет: Низкий
Серьёзность: Низкая
Статус: Улучшение

Описание

Метод реализован через index() + try/except, что усложняет код.

Текущая реализация
try:
    res = string.index(symbol) > -1
except ValueError:
    pass
Проблема

Лишняя обработка исключений

Менее читаемо

Менее эффективно

Ожидаемое решение

Использовать:

return symbol in string
Влияние

Не критично, но ухудшает читаемость и поддерживаемость.

DEF-007

Название: capitalize() меняет регистр всей строки

Приоритет: Средний
Серьёзность: Средняя
Статус: Новый

Описание

Метод использует string.capitalize(), который:

делает первую букву заглавной

делает остальные символы строчными

Шаги воспроизведения
utils.capitalize("sKYpRO")
Фактический результат

"Skyporo"

Возможная проблема

Если ожидалось изменение только первой буквы, поведение некорректное.

Рекомендация

Если бизнес-логика требует сохранить регистр остальных букв:

string[0].upper() + string[1:]
